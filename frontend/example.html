Cities Generator Dropdown - Integration Examples

Method 1: Web Component (Most Portable)

Works in any platform - WordPress, Drupal, Liferay, Joomla, etc.

<script src="https://unpkg.com/react@18/umd/react.production.min.js"></script>
<script src="https://unpkg.com/react-dom@18/umd/react-dom.production.min.js"></script>
<script src="./dist/cities-generator.umd.js"></script>

<!-- Using default URL -->
<cities-dropdown country="it" placeholder="Select location..."></cities-dropdown>

<!-- Using custom base URL (automatically appends /it.json) -->
<cities-dropdown country="it" data-url="https://example.com/cities" placeholder="Select location..."></cities-dropdown>

Method 2: Standalone Bundle (Zero Dependencies)

Includes React - no external dependencies needed.

<div id="my-dropdown"></div>
<script src="./dist/cities-generator-standalone.iife.js"></script>
<script>
  // Using default URL
  CitiesGenerator.render('#my-dropdown', {
    country: 'it',
    placeholder: 'Select location...',
    onSelect: (node) => console.log(node)
  });
  
  // Using custom base URL (automatically appends /it.json)
  CitiesGenerator.render('#my-dropdown', {
    country: 'it',
    dataUrl: 'https://example.com/cities',
    placeholder: 'Select location...',
    onSelect: (node) => console.log(node)
  });
</script>

Method 3: JavaScript API

Programmatic rendering with full control.

// Using default URL
CitiesGenerator.render('#container', {
  country: 'it',
  placeholder: 'Select location...',
  onSelect: (node) => console.log(node)
});

// Using custom base URL (automatically appends /it.json)
CitiesGenerator.render('#container', {
  country: 'it',
  dataUrl: 'https://example.com/cities',
  placeholder: 'Select location...',
  onSelect: (node) => console.log(node)
});

Integration Notes

  • WordPress: Upload files to theme, enqueue scripts in functions.php
  • Drupal: Add to .libraries.yml, attach in template
  • Liferay: Include in JSP/FreeMarker with CDN URLs
  • Joomla: Add scripts to template index.php
  • Any Platform: Use Method 1 (Web Component) for maximum compatibility

See INTEGRATION.md for detailed platform-specific instructions.
